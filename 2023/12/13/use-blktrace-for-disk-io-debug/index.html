use blktrace for disk io debug

John Doe
2023-12-13

在工作中经常会遇到磁盘的IO很高的情况，一方面需要定位到底是那些程序在频繁的访问磁盘，另外一方面需要指导这些访问磁盘的程序究竟在访问那些文件，经过一轮调查，可以考虑使用blktrace来对块设备进行分析排查。具体如下

  1. 利用blktrace 分析块设备的io情况，blktrace 默认情况下会对每个cpu上对块设备的访问做记录，每个cpu对应一个文件

  2. 使用debugfs 定位具体操作的块设备所在的文件地址, 需要将块设备的扇区编号转换成块编号（一个扇区512B，一个块4096B），转换的公式为： blockNum = sectorNum * 512 / 4096, 根据块信息拿到inodenumber，通过ncheck获取对应的文件路径
